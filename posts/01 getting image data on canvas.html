Day 1 : Getting image data on canvas

posted on 2015-12-01

Hi there! This will be a series of blog posts exploring image processing with browser and HTML5 canvas element. Before we start dive into filers and effects, let's start by looking at how image data is handled on canvas.

Loading an image to browser

As web engineer, we take it for granted <img src="image.jpg"> displays an image in a browser because it is a "digital photo". But how does the image look like when you are dealing with image with JavaScript?

Use Image contractor to create image element

The way to create HTMLImageElement (without writing HTML tag) is to use a Image constructor. You can create new Image instance, then add src attribute to the instance much like you type <img src="image.jpg">. At this point, img object you created has same properties as HTMLImageElement, in the sample you can see width and height properties are console logged.

JS Bin on jsbin.com

Draw on canvas

Let's try to get not only properties but also the actual data of this image. Canvas element in HTML5 has nifty method getImageData which provides pixel by pixel information about an image.

Let's start by creating Canvas element. There is no constructor like we used for Image, so we'll have to do document.createElement('canvas') much like you create other HTML element in JavaScript.

So let's draw an image into this newly created canvas element. It may feel natural to call method on the element itself like canvas.drawImage()...however, in canvas element rendering layer - a layer that handles drawing on browser - is provided as separate thing called context.

what is "context" ?

Context is a layer that handles "what is drawn on where". You can specify how you want to send drawing instruction (called contextType). In this post, we are going to use '2d' context type - which let us specify x and y position of an object we want to draw. Other context type is webgl which is used to draw 3D object. You can get 2d context by calling getContext('2d') on canvas element.

drawImage and getImageData

Now we are ready to draw our image into canvas using our 2d context. Since we have Image object, we can use drawImage() method on 2d context to draw image on our canvas element. Once the image is in canvas, we can call getImageData() on 2d context to see data underneath the canvas.

JS Bin on jsbin.com

As you can see in the example, context returned ImageData object which contains width, height, and data of the canvas. We'll use this data for our image processing expiration from now on.
